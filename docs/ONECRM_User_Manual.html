How to use this manual

This is a user manual: it tells you what to click and in what order. Bold labels like Add Lead match buttons and menu names in the app.

What you see depends on your role, permissions, and which modules your organisation has enabled. If a menu is missing, ask your Global Admin.

1. Sign in and get around

1.1 Choose the right portal

I am…Open this pageAfter login you go to
Staff (admin, counsellor, marketer, HR)/loginDepends on role (often Marketing or HR)
Agency agent / freelancer/agent-login/agency-crm/dashboard
Student / applicant/student-loginApplicant portal (after policy)

1.2 Staff sign in

  1. Open the staff login page. Heading: Sign in.
  2. Enter Email and Password.
  3. Click Sign in.
  4. Optional links: Forgot password?, Register, Use student login, Agent login.

1.3 Agent sign in or register

  1. Open /agent-login → Agent sign in.
  2. To register: click Register here.
  3. Fill Personal details (Full name, Email, Phone, Password, Confirm password).
  4. Fill Agency details (Agency name required; optional referral code, address, city, country).
  5. Click Register as agent.
  6. Wait for admin approval. Message: Registration received. An admin will approve your account before you can sign in.
  7. After approval, sign in with Sign in as agent.
Only agent roles can use the agency portal. Other roles see: This portal is for agency agents only…

1.4 Student sign in

  1. Open /student-login.
  2. Enter the email/password your counsellor created for you.
  3. Click Sign in.
  4. If asked, set a new password, then accept Terms & conditions with I accept.

1.5 Navigation (staff / agent)

  • Left sidebar — modules (Marketing, Student CRM, Agency CRM / Agent Portal, Human Resource, Admin & Settings, Allied Services).
  • Top bar — page title, notification bell, Add Lead, Log out.
  • If access is blocked you see Access Denied with Go Back.

Agents see Agent Portal instead of Agency CRM, with labels such as My Students, My Commissions, and Referral & Branding.

2. Passwords, profile, and notifications

2.1 Forgot / reset password

  1. On any login page click Forgot password?.
  2. Enter Email → Send reset link.
  3. Open the email link → enter New password and Confirm password → Reset password.
  4. You are sent to staff /login; students/agents should then open their own portal login.

2.2 Change password while signed in

  1. Open /change-password (forced on first login, or from profile).
  2. Enter Current password, New password, Confirm password.
  3. Meet the rules shown on screen (8+ characters, number, special character).
  4. Click Continue.

2.3 Notifications

  1. Click the bell in the top bar.
  2. Use Mark all read, open a row to follow its link, or delete with the trash icon.
  3. View all opens the full inbox at /notifications (filters: all / Unread only).

2.4 Staff profile

  1. Open /profile.
  2. Upload or replace photo (JPG/PNG/WebP, max 5MB).
  3. Review Email, Phone, Role (read-only).
  4. Click Change password if needed.

3. Marketing — leads, campaigns, analytics

Sidebar: Marketing.

Lead Management access: the page currently allows only SUPER_ADMIN, GLOBAL_ADMIN, and COUNSELLOR. Counsellors only see leads assigned to them. Rating and counsellor assignment dropdowns are editable for admins.

3.1 Create a lead

  1. Go to Marketing → Lead Management, or click top-bar Add Lead.
  2. Click Add Lead.
  3. Fill Full Name *, Email Address *, optional phone, country, preferred course/country, source, counsellor, rating (HOT/WARM/COLD/MAYBE), remarks.
  4. Click Create Lead.

3.2 Find, filter, export, bulk upload

  • Search: Search leads...
  • Filters: date range, source, rating, stage, counsellor → Reset Filters to clear.
  • Upload — choose .xlsx / .xls / .csv.
  • Export — downloads a CSV of the leads you can see.

3.3 Update rating, stage, or counsellor

  1. In the table, change Lead Status (rating) or Stage (New / Contacted / Qualified / Proposed / Converted / Lost).
  2. In Assigned To, pick a counsellor or Unassigned (admins).

3.4 Log an activity

  1. Click the lead name to open the activity panel.
  2. Review Interaction History.
  3. Choose channel: NOTE, CALL, EMAIL, SMS, WHATSAPP, MEETING.
  4. For NOTE/CALL, type details and send; other channels run a quick action then refresh history.

3.5 Create student login from a lead

  1. In the row Action column (or activity panel), click Create Login.
  2. Student receives a welcome email; badge becomes Login Active.

This creates the portal login only. To create the full Student CRM record + application, use Applications → Create login + application (Section 4).

3.6 Campaigns

  1. Open Marketing → Campaigns.
  2. Click Create Campaign (modal: Launch Campaign).
  3. Enter name, broadcast type (Email / Social / Google Ads), status, dates, audience, and type-specific settings.
  4. Click Save & Launch (or Save & Launch Ad).
  5. Use row menu ⋯ for Edit details, Launch Campaign, or Delete.
  6. To enroll more leads: open performance drawer → Enroll … Leads → select → confirm.

3.7 Marketing analytics

  1. Open Marketing → Marketing Analytics.
  2. Review KPI cards and panels: Agency Funnel, Lead Source Performance, Monthly Lead Trend, Counsellor Performance.

4. Student CRM — students, applications, visas

Sidebar: Student CRM. Creating/editing needs manage permission.

4.1 Create and edit a student

  1. Go to Student Management.
  2. Click New student → enter Full name *, Email *, Phone, DOB, Preferred country → Create.
  3. Select the student in the left list (search: Search name or email...).
  4. Use tabs: Personal | Study plan | Education | Exams | Checklist | Applications.
  5. Edit fields → click Save profile (not shown on Checklist/Applications, or when enrolled).

Study plan universities

  1. On Study plan, under Universities, click Add university.
  2. Set Destination, University, Course, Intake (autosaves).
  3. Select radio Use for new application when ready.

Enrolment lock

  • Mark enrolled → profile mostly read-only, badge Enrolled.
  • Unlock profile to edit again.

4.2 Convert a marketing lead into an application

  1. Open Applications.
  2. Expand Leads ready for applications.
  3. Per lead click Create login + application (temp password often shown in toast, e.g. Welcome@123).
  4. Or click Import all leads and confirm the batch.

4.3 Create an application

  1. From Applications list: New application (or from student profile New application).
  2. Choose student if prompted; set country, university, course, intake, deadline, counsellor, notes.
  3. Save — you open the application detail page.

4.4 Work an application (detail page)

Header shows code, stage, university/course, Advance stage, and link Student profile.

Stages: Draft → Documents pending → Submitted to university → Under review → Offer received → Offer accepted → Visa process → Enrolled.

  • At Offer received: Mark accepted / Mark rejected.

4.5 Filter and bulk-assign applications

  1. Use stage chips (All, Draft, Docs pending, … Enrolled) and search.
  2. Select rows → choose counsellor → Assign selected.

4.6 Visa management list

  1. Open Visa management.
  2. Search student or university.
  3. Click Open to jump to that application’s Visa tab.

4.7 CRM Settings

  1. Open CRM Settings.
  2. Universities & courses: filter by country → Add university → fill name/country/city.
  3. Document checklist templates: choose Country (optional University) → Load built-in defaults or Add document → Save template.

5. Agency CRM & Agent Portal

5.1 Approve a new agent (admin)

  1. Open Admin & Settings → Users.
  2. Open the Pending Agents tab.
  3. Review Agency Details.
  4. Click Approve (or Reject → confirm deactivate).

5.2 Agent onboarding

  1. Agent opens My Profile & Onboarding.
  2. Under upload step: Upload ID / business docs (at least one file) → Submit documents.
  3. Click Sign agreement.
  4. Wait while admin verifies, approves, and activates.

Stages: Registered → Documents submitted → Agreement signed → Verified → Approved → Active.

Admin completes activation

  1. Open Agency Management.
  2. Select the agency → under Partner documents use View, then Verify or Reject.
  3. Use Verify onboarding → Approve partner → Activate (or status buttons Mark Verified / Approved / Active).

5.3 Create an agency (admin)

  1. Add agency → fill Agency name, contact, login email, initial password, commission rate, status, services, notes.
  2. Create agency.
  3. Edit later → Save changes.
  4. Set Capabilities (Can pay student fees, Can message university, Can manage applications, Can view commissions) → Save capabilities.

5.4 Referrals and students

Agent: open My Students.

  1. Use Get referral link if needed.
  2. Switch tabs My students / Applications; filter by name/email/code.
  3. Click student name or Open.
  4. On student detail: Attach to upload docs; View file; Pay now only if fee payment capability is enabled.

Staff assign a student to an agency:

  1. Open Students & Referrals → Assign student.
  2. Choose agency → search student by name/email → pick one → optional notes → Confirm assignment.
Until referral-link capture is fully wired end-to-end, Assign student is the reliable way to attribute a student to an agency.

5.5 University directory, communications, branding

  • Universities — search name/country → Search → view POC email/phone.
  • Notifications (agent) / Communications (staff) — read announcements; log Call/Email/Meeting/Note; admins can Publish announcement and Send broadcast.
  • Referral & Branding — copy referral link; edit tagline/colours/logo/website → Save branding.

5.6 Commissions and payouts

Agent: My Commissions — filter statuses; review statement totals (Total / Paid / Pending).

Admin:

  1. Add commission → agency, amount, period, description → Save.
  2. Pending rows: Verify & approve or Reject.
  3. Select approved rows → Create payout batch.
  4. On Payouts tab: Start processing → Mark paid (or Cancel).
  5. With an agency filter, manage Commission rules (Country, University, %/Fixed, Amount → Add rule).

6. Human Resources

Sidebar: Human Resource. Needs the HR module enabled and VIEW_HR.

Users with only self-service access are redirected to My HR. Operator screens (directory, recruitment, payroll run, leave policies) need extra permissions such as MANAGE_EMPLOYEES, MANAGE_LEAVE, or MANAGE_PAYROLL.

6.1 Overview and My HR

  1. Overview (/hr) — use Clock in / Clock out on the Today card; open KPI tiles or Review leave / Review attendance when shown.
  2. My HR (/hr/me) — personal attendance, leave balance, recent leave, and payslips. Shortcuts: View full attendance →, Apply for leave →, View all → (payslips).

6.2 Employee Directory

  1. Open Employee Directory. Search by name/email/department/id; filter status: All / Active / On leave / Resigned / Terminated.
  2. Add employee — Full name *, Email *, Employee ID *; optional phone, department, designation, location, joining date, access role → Add employee.
  3. Bulk import — paste spreadsheet rows → validate rows → import verified accounts.
  4. Click View on a row. On Profile, edit fields (including employment status / exit reason) → Save profile.
  5. On Documents: choose type (Offer letter / ID proof / Contract / Other) → Choose files → Upload.
  6. Quick role change from the list: Edit role → save changes.

6.3 Recruitment tracker

Open Recruitment. Tabs: Jobs & pipeline, Interviews, Offer letters, Onboarding.

Jobs & pipeline

  1. Post job → fill title, department, location, salary, hiring manager, dates, requirements, description, employment type, publish status → Create posting.
  2. Draft jobs: Publish. Open jobs: Pause / Close. Edit with Edit.
  3. Add candidate → job, name, email, phone, optional resume → Add candidate.
  4. Move candidates with the stage dropdown/chips (application → screening → hr_round → tech_round → manager_round → offer_generation → onboarding → hired).
  5. Candidate actions: Schedule interview, Create offer, Reject, Hire.

Interviews, offers, onboarding

  1. Schedule Interview — candidate, job, round, date/time, duration, interviewers, meeting link, type → Schedule. Later: Reschedule, Feedback, Cancel.
  2. Generate an offer — candidate, salary, joining date, expiry, template, optional conditional offer. Then Preview / Print; Draft → Send; Sent → Accept / Reject / Mark expired.
  3. Accepting an offer creates the employee record, starts onboarding, and marks the candidate Hired.
  4. Onboarding: New Checklist → employee + template + start date → Create Checklist; tick items and upload document files as needed.

6.4 Attendance

  1. Open Attendance.
  2. Clock in / out tab — big Clock in / Clock out buttons and today’s summary.
  3. My history — pick month/year.
  4. Regularizations — Request regularization (date, clock in/out type, time, reason) → Submit request.
  5. Managers: Team attendance and Admin setup (process biometric logs; register devices; whitelist IP networks).
Clock-in needs your login linked to an employee record. If not linked, the screen asks you to contact HR.

6.5 Leave

  1. Open Leave. Tabs: My leave, Approvals, Policies (managers).
  2. Apply: leave type, From, To, Days, Reason → Submit request. Cancel with ✕ while still pending/manager-approved.
  3. Approvals are two-stage: manager uses Approve (to HR); HR uses Final approve or reject.
  4. Policies: entitlement plans, leave-type rules, assigned employees; holiday calendar (Add holiday); leave categories.

6.6 Performance

  1. Open Performance — tabs Reviews / KPIs.
  2. Reviews: choose Monthly/Weekly period → Calculate reviews; set status (Self-Review / Manager Review / Calibrated / Completed).
  3. KPIs: add/edit definitions (role, name, target, unit, frequency) and record metrics.

6.7 Payroll

  1. Open Payroll — tabs Payroll run / Deductions.
  2. Salary Structures: edit basic/allowances/incentive/revenue share/deductions → Deploy Structure.
  3. Execute Batch: pick Month/Year → Execute Ledger Cycle (confirm).
  4. Statements ledger: View a payslip → Export / Print Statement.
  5. Deductions: Add Deduction (leave days, tax, other) → Save.

Users with only VIEW_OWN_PAYSLIP see their own payslips, not full payroll admin.

7. Admin & Settings

Users is limited to SUPER_ADMIN / GLOBAL_ADMIN. Editing roles needs MANAGE_ADMINS.

7.1 Users

  1. Open Admin & Settings → Users.
  2. Tabs: All Users, Pending Agents, Pending Students. Search and filter by role/status.
  3. Create User — First/Last name, Email, Phone; Role & Status; Module Access checkboxes per module → Create User (credentials emailed).
  4. Edit with the pencil → update role/status/module access → Update Access.
  5. Pending Agents: Approve or Reject.
  6. Pending Students: pick counsellor → Assign.

7.2 Roles & Permissions

  1. Open Roles & Permissions.
  2. Click a role to expand permission groups; toggle checkboxes.
  3. Click Save permissions (or Reset all to defaults).
  4. Assignment column: + Assign a member, or move them with ✕ → Move to…

7.3 Allied Services

  1. Open Allied Services.
  2. Click a service card to open the external site in a new tab.

8. Applicant (student) portal

Sign in at /student-login. Sidebar: Applications, Payments, Profile.

8.1 Accept policy (first time)

  1. On Terms & conditions, click I accept.

8.2 Applications home

  1. Open Applications.
  2. Review overall progress and quick actions: Upload document, View applications, Payments, Contact counsellor.
  3. Use search and stage filters; click Open application.

8.3 Application detail

  1. Follow workflow steps (Upload documents → Pay fee → Counsellor review → Offer → Visa & enrolment).
  2. Upload/replace pending documents.
  3. When offer is ready: View offer letter, Accept offer, or Decline offer (confirm).
  4. When fees are due and documents are complete: Pay now (Razorpay).
  5. Upload visa documents when requested.

8.4 Payments

  1. Open Payments.
  2. Filter All / Paid / Pending / Failed.
  3. Use Pay on application or open the application to pay.
  4. Open Receipt for paid items.

8.5 Profile

  1. Open Profile.
  2. Upload/replace photo; other fields are counsellor-managed.
  3. Use Contact counsellor when shown.

9. Super Admin (platform operators)

Only SUPER_ADMIN — /super-admin. No normal tenant sidebar; this is cross-tenant.

9.1 Tenants list

  1. Review Name, Slug, Status, Primary admin, Users, Enabled modules.
  2. Click Manage → for a tenant, or use top-right Audit log / + Onboard tenant.

9.2 Onboard a tenant

  1. Click + Onboard tenant.
  2. Organization: Name (Slug auto-generates).
  3. Modules: tick Human Resources, Marketing, Student CRM, Agency CRM, Admin Settings.
  4. Primary admin: Full name, Email, Password (min 8), optional Phone.
  5. Click Create tenant.

9.3 Manage a tenant

  1. Change status: ACTIVE / SUSPENDED / ARCHIVED.
  2. Primary admin card: Reset password → confirm → copy the new password once (shown only once) → Dismiss.
  3. Toggle modules Enabled/Disabled (takes effect immediately).

9.4 Audit log

  1. Open Audit log / /super-admin/audit.
  2. Review actions such as Created tenant, Updated tenant, Set modules, Reset admin password.
